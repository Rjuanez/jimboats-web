# Workflows

## Purpose

Workflows describe behavior that crosses module boundaries.

Create a workflow only when a process cannot be clearly owned by one module.

## Workflow Candidates

- [Booking Checkout](booking-checkout.workflow.md)
- [Publish Localized Experience Page](publish-localized-experience-page.workflow.md)
- Upload media and expose public variants.
- Submit contact or booking lead.

## Naming

Workflow files use:

- `<name>.workflow.md`
