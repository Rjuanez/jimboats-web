# Media Processing Job

> File name: `media-processing-job.entity.md`

## Purpose

Represents a worker job that generates optimized image variants for a media
asset.

The domain should make the processing lifecycle clear. The implementation can
choose the concrete worker technology later.

## Identity

- `mediaProcessingJobId`: stable internal identifier.

## Attributes

- `mediaAssetId`: asset being processed.
- `status`: pending, processing, succeeded, failed, or cancelled.
- `requestedVariants`: variant keys to generate.
- `attempts`: number of processing attempts.
- `lastError`: last failure reason.
- `createdAt`: job creation time.
- `startedAt`: optional start time.
- `finishedAt`: optional finish time.

## Invariants

- Job must reference an existing media asset.
- Succeeded jobs must have generated all requested required variants.
- Failed jobs must record a reason.
- Processing should be idempotent for the same asset and requested variant set.

## State

- `PENDING`: waiting for worker.
- `PROCESSING`: worker is generating variants.
- `SUCCEEDED`: variants were generated.
- `FAILED`: processing failed.
- `CANCELLED`: job was cancelled before completion.

## Behavior

- Start processing.
- Mark succeeded and attach generated variants.
- Mark failed with reason.
- Retry failed processing.
- Cancel pending job.

## Relationships

- `MediaAsset`: source asset and resulting status.
- `MediaVariant`: output records generated by successful processing.

## Domain Errors

- `MediaProcessingJobInvalid`
- `MediaProcessingJobAlreadyRunning`
- `MediaProcessingJobFailed`
