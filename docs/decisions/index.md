# Decisions

## Purpose

Decisions record product-domain and architecture choices that should not be
rediscovered later.

Use decisions for tradeoffs that affect module boundaries, publication rules,
SEO/GEO behavior, storage, integrations, or implementation constraints.

## Naming

Decision files use:

- `0001-<decision>.md`
